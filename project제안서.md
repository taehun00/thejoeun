
## ✨기본 콘셉트 예시 - 반려동물 사료 분석 기반 



| 단계 | 기술 스택                             | 주제                      | 핵심 기능          |
| -- | --------------------------------- | ----------------------- | -------------- |
| 1  | JSP + Oracle                      | 사료/간식 DB CRUD           | 제품 등록/조회       |
| 2  | Spring MVC + MyBatis              | 관리자용 콘텐츠 관리             | 성분별/건강별 분류     |
| 3  | Spring Boot + JPA                 | 반려동물 프로필 기반 맞춤 추천       | 활동량·알러지 필터     |
| 4  | Node + React                      | 리뷰 커뮤니티                 | 후기, 사진, 평점     |
| 5  | Spring Boot + React + JWT + Redis | 통합 건강 먹거리 추천 플랫폼        | 사용자 인증 + 추천 피드 |
| 6  | Flutter                           | 모바일 버전 “PET-EAT Mobile” | 사료 검색/리뷰/피드 연동 |




## 🐾 **1단계 JSP + Oracle 분업 설계 (CRUD 중심)**

**공통 주제:** 반려동물 맞춤 먹거리 관리 시스템 (PET-EAT v1)

각자 맡은 기능은 모두 독립된 CRUD이지만, 나중에 연동하면 자연스럽게 “추천 로직”이나 “리뷰 시스템”으로 발전 가능.

---

### 👤 1번 — 제품 정보 관리 (FOOD CRUD)

> **역할:** 데이터의 중심축. 전체 시스템의 기본 테이블 담당.

* **주제:** 반려동물 사료/간식 정보 등록 시스템
* **기능:**

  * 제품명, 브랜드, 유형(사료/간식), 대상(강아지/고양이), 칼로리, 주요 성분, 가격 입력
  * 등록 / 수정 / 삭제 / 조회
* **DB 테이블:** `FOOD_PRODUCT`
* **포인트:** 기본 SELECT + INSERT + UPDATE + DELETE 구현 완성도 중점

---

### 👤 2번 — 반려동물 프로필 관리 (PET CRUD)

> **역할:** 추천과 매칭의 기반이 되는 개체 데이터 담당.

* **주제:** 반려동물 프로필 등록 및 관리 시스템
* **기능:**

  * 이름, 품종, 나이, 몸무게, 알러지, 활동량 입력
  * 보호자와 연결된 여러 반려동물 관리
* **DB 테이블:** `PET_PROFILE`
* **포인트:** 보호자(회원) 외래키로 연동 가능하게 설계

---

### 👤 3번 — 보호자(회원) 관리 (USER CRUD)

> **역할:** 전체 시스템의 사용자 데이터 기반 담당. 로그인까지는 불필요, 단순 회원 등록 중심.

* **주제:** 반려동물 보호자 회원정보 등록 시스템
* **기능:**

  * 이름, 연락처, 이메일, 주소
  * 회원 등록 / 수정 / 삭제 / 조회
* **DB 테이블:** `USER_INFO`
* **포인트:** 나중에 JWT 로그인 확장 시 핵심 테이블로 발전 가능

---

### 👤 4번 — 리뷰 & 평가 관리 (REVIEW CRUD)

> **역할:** 제품 피드백 데이터를 담당. 후속 단계의 커뮤니티 기능 기초.

* **주제:** 반려동물 먹거리 리뷰 관리 시스템
* **기능:**

  * 제품 ID 참조, 회원 ID 참조
  * 평점(1~5), 한줄평, 작성일
  * CRUD (리뷰 수정/삭제/조회/등록)
* **DB 테이블:** `FOOD_REVIEW`
* **포인트:** 외래키 관계(회원, 제품) 실습에 최적

---

### 👤 5번 — 영양 성분 & 카테고리 관리 (NUTRIENT CRUD)

> **역할:** 추천 알고리즘의 기반 데이터 담당. 제품 속성 정의자.

* **주제:** 영양성분 및 카테고리 관리 시스템
* **기능:**

  * 성분명(단백질, 지방, 섬유질 등), 설명, 권장 섭취량
  * 특정 성분이 포함된 제품 리스트 조회
* **DB 테이블:** `NUTRIENT_INFO`
* **포인트:** 나중에 JPA 단계에서 추천 매칭 로직으로 활용 가능

---

## 🔗 ERD 개념 연결 예시 (단계별 확장 고려)

```
USER_INFO (회원) 
   ↓ (1:N)
PET_PROFILE (반려동물)
   ↓ (1:N)
FOOD_REVIEW (리뷰)
   ↑ (N:1)
FOOD_PRODUCT (제품)
   ↓ (N:M)
NUTRIENT_INFO (성분)
```

## ⚙️ 기술 균형 가이드

| 팀원 | 기능   | DB 연습 포인트     | JSP 포인트     |
| -- | ---- | ------------- | ----------- |
| 1  | 제품   | 기본 CRUD       | 리스트/상세보기    |
| 2  | 반려동물 | 다중 등록, 조건검색   | 폼처리         |
| 3  | 회원   | 기본 + PK/FK 설계 | Join 실습     |
| 4  | 리뷰   | FK, 정렬/평점 계산  | 조건부 출력      |
| 5  | 영양   | 관계형 설계        | 검색 + 필터 JSP |

## 💬 확장 시 자연스러운 다음 단계

* 2단계(Spring MVC): “관리자용 콘텐츠 관리” 시, 이 5개 테이블 그대로 MyBatis로 전환 가능.
* 3단계(Spring Boot): 각 기능을 서비스/리포지토리 레이어로 세분화하면서 통합 추천 구현.
