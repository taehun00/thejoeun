**Paw + Project = 반려동물을 위한 개발 여정**

## 🐶😺 Why Pawject?
사료 선택은 생각보다 어렵습니다.  
복잡한 라벨, 다양한 브랜드, 건강 상태에 따른 수많은 선택지…

> **“라벨을 일일이 읽지 않아도, 원하는 사료를 쉽게 찾을 수 없을까?”**

이 질문에서 Pawject가 시작되었습니다.  
우리는 **신뢰할 수 있는 데이터**, **직관적인 UI**, **사용자 흐름 중심의 설계**를 통해  
보호자의 선택을 돕는 서비스를 만들고자 했습니다.

---

## 🚀 Project Overview
- **프로젝트명:** 반려동물 건강·종합 사료 플랫폼  
- **개발 방식:** 기능을 순차적으로 확장하며 **v1~v4 버전업 고도화**
- **구성:** Spring 기반 백엔드 + React 전환 진행(v4)

---

## 🐾 버전업 진행(v1~v4)

| 버전 | 핵심 내용 |
|------|----------|
| **v1** | 핵심 게시판 기능 구현 및 기본 데이터 구조 설계(프로토타입) |
| **v2** | 기능 고도화: 게시판 확장, 검색/페이징 도입, 유지보수성 개선 |
| **v3** | 서비스 확장: 커뮤니티 기능 강화 및 외부 API 연동 기능 추가 |
| **v4 (진행중)** | 아키텍처 고도화: Spring+React 구조 전환, UI 리뉴얼 및 신규 기능 추가 |

## 📌 주요 기능

### 회원/보안
- 회원가입/로그인
- 권한 기반 접근 제어(Spring Security)

### 사료 데이터
- 관리자 사료 데이터 관리
- 조건 기반 검색/필터링
- 페이징 처리

### 커뮤니티
- 리뷰 게시판(사료 리뷰 작성)
- 고객센터(문의/답변, 사용자/관리자 화면 분리)
- 체험단 게시판(v4 신규)

### 건강/운동
- 건강 정보 게시판(UI 카드형 리뉴얼)
- 운동 기록 공유 기능(v4 SNS형 개편 진행)

### API 연동(v3)
- AI 기반 검색 필터 추천(선택 기능)
- 리뷰 문장 다듬기(선택 기능)
- OCR 라벨 출력(선택 기능)
- 1:1 채팅 연동(선택 기능)
> 비용 제약으로 일부 기능은 우선순위를 정해 단계적 도입
